# Frontend: i18n (next-intl)

This repo uses `next-intl` for translations.

## Where translations live

- Messages: `src/i18n/locales/en/translations.json`
- Locale config: `src/i18n/config.ts`
- Request config (Next.js plugin entry): `src/i18n/request.ts`

Currently, only `en` is configured:

- `defaultLocale = 'en'`
- `locales = ['en']`

## How itâ€™s wired into Next.js

- `next.config.ts` wraps config with `next-intl/plugin` pointing at `./src/i18n/request.ts`
- `src/app/layout.tsx` loads messages via `getMessages(defaultLocale)` and provides them via `NextIntlClientProvider`

## How to add a new locale

1. Add a new translations file:
   - `src/i18n/locales/<locale>/translations.json`
2. Update:
   - `src/i18n/config.ts` (`locales` array + `defaultLocale` if needed)
3. Ensure any routing/locale selection strategy you want is implemented in `src/i18n/request.ts`
